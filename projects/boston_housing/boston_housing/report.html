Implementation: Define a Performance MetricÂ¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

# Import 'r2_score'
from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_predict) 
    
    # Return the score
    return score

Question 7 - Grid SearchÂ¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: Most learning algorithms have hyper-parameters that can be tuned for a given learning challenge to give the optimal result. E.g. for decision trees possible hyper-parameters are the maximum depth of the tree and the strategy for finding the best split. If the best combination of hyper-parameters can't be derived in an analytical way, then usually a brute-force search strategy is used. This search can be very time-consuming, because the search space is defined by all tupels of possible combinations of hyper-parameters. One strategy is a systematic, exhaustive search along the points of a grid. The points of the grid are defined by all possible combinations of hyper-parameters from given finite sets. For numeric hyper-parameters, the possible value sets are usually sampled in an equidistant fashion. Grid search then goes systematically along the points of the grid, trains the learning algorithm with the candidate hyper-parameters and scores the resulting model with a given scoring function. The output is a score for each point on the grid, and the model with the highest score can then be accepted, or the grid search can be run again with a finer mesh in a smaller space around the best combination of hyper-parameters.

Question 8 - Cross-ValidationÂ¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: k-fold cross-validation is the process of dividing the dataset into k random subsets, called folds. A fixed combination of model paramters is chosen and the model is then trained with (k-1) of these folds and tested against the fold that was held back from training. The training and testing procedure is repeated with each fold as the test set and the final score of the fixed model parameters is taken as the average of all scores. This can be repeated for the next parameter combination on the search grid. With only one training and test set we run the risk that a parameter combination fits it perfectly just by chance. By shuffling the dataset around, this risk is very low and the combination of grid search and cross validation should give accurate results.

# Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    #cv_sets = ShuffleSplit(X.shape[0], n_splits = 10, test_size = 0.20, random_state = 0)
    rs = ShuffleSplit(n_splits=10, test_size=.20, random_state=0)
    cv_sets = rs.get_n_splits(X, y)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': np.linspace(1, 10, 10)}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search object
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
